# Security Policy

## Supported Versions

This repository is actively maintained on the `main` branch.

If you need a security fix, please update to the latest commit on `main`.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, report privately using one of these options:

1. **GitHub Security Advisories** (preferred)
   - Go to: Repository → **Security** → **Advisories** → **New draft security advisory**
2. Email: **security@your-domain.example** (replace with your real address)

When reporting, include:
- A clear description of the issue
- Steps to reproduce (PoC if possible)
- Impact (what an attacker can do)
- Affected versions/commit

We will:
- Acknowledge receipt within 72 hours
- Provide a status update as soon as possible
